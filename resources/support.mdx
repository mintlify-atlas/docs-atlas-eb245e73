## Before Asking for Help

To get the fastest resolution, please:

<Steps>
  <Step title="Check the documentation">
    Search our [documentation](/introduction) for your question. Common topics:
    - [Installation issues](/quickstart)
    - [Troubleshooting](/resources/troubleshooting)
    - [FAQ](/resources/faq)
    - [Feature guides](/guides/creating-your-first-chat)
  </Step>

  <Step title="Search existing issues">
    Someone may have already reported your issue:
    - [GitHub Issues](https://github.com/21st-dev/1code/issues)
    - [GitHub Discussions](https://github.com/21st-dev/1code/discussions)
    - Discord search (use the search bar)
  </Step>

  <Step title="Gather information">
    Before asking for help, collect:
    - **OS and version** (macOS 13.4, Windows 11, Ubuntu 22.04, etc.)
    - **1Code version** (check in Settings → About)
    - **Build type** (open source build or 1code.dev subscription)
    - **Error messages** (full text or screenshots)
    - **Steps to reproduce** (what you did before the issue)
    - **Expected vs actual behavior**
  </Step>
</Steps>

## Reporting Bugs

When reporting bugs, include:

### Required Information

1. **Clear description** of the problem
2. **Steps to reproduce** the issue:
   ```
   1. Open 1Code
   2. Create a new chat
   3. Click X button
   4. Error appears
   ```
3. **Expected behavior**: What should happen
4. **Actual behavior**: What actually happens
5. **Environment**:
   - OS and version
   - 1Code version
   - Build type (open source or 1code.dev)

### Optional but Helpful

- **Screenshots or videos** showing the issue
- **Logs** (if applicable):
  - macOS: `~/Library/Logs/1Code/`
  - Linux: `~/.config/1Code/logs/`
  - Windows: `%APPDATA%/1Code/logs/`
- **Console errors** (open DevTools with Cmd+Option+I or Ctrl+Shift+I)
- **Workarounds** you've tried

### Where to Report

<CardGroup cols={2}>
  <Card title="Critical Bugs" icon="exclamation-triangle" href="https://github.com/21st-dev/1code/issues/new">
    Data loss, crashes, security issues - create a GitHub issue immediately
  </Card>
  <Card title="Minor Bugs" icon="bug" href="https://discord.gg/8ektTZGnj4">
    UI glitches, small errors - report in Discord first
  </Card>
</CardGroup>

## Requesting Features

We love hearing your ideas! Here's how to request features:

<Steps>
  <Step title="Check if it already exists">
    Search:
    - [Open issues](https://github.com/21st-dev/1code/issues?q=is%3Aissue+is%3Aopen+label%3Aenhancement)
    - [Discussions](https://github.com/21st-dev/1code/discussions)
    - [Roadmap](https://github.com/21st-dev/1code/projects) (if available)
  </Step>

  <Step title="Start a discussion">
    Before creating an issue, [start a discussion](https://github.com/21st-dev/1code/discussions/new) to:
    - Gauge community interest
    - Get feedback on the idea
    - Refine the proposal
  </Step>

  <Step title="Create a feature request">
    If there's strong interest, [create an issue](https://github.com/21st-dev/1code/issues/new) with:
    - **Problem statement**: What problem does this solve?
    - **Proposed solution**: How should it work?
    - **Alternatives considered**: Other approaches you thought of
    - **Use cases**: When would you use this?
    - **Mockups** (if applicable): Screenshots or sketches
  </Step>
</Steps>

<Tip>
  Features that solve common problems and align with 1Code's vision are more likely to be implemented.
</Tip>

## Emergency Support

For critical production issues (Max subscribers only):

- **Email:** support@21st.dev with "URGENT" in subject
- **Response time:** 2-4 hours during business hours

**What counts as critical:**
- Data loss
- Security vulnerabilities
- Complete service outage
- Breaking bugs in production

**What doesn't:**
- Feature requests
- Questions
- Minor bugs
- Enhancement ideas

## Community Guidelines

When seeking help:

- **Be respectful** to everyone
- **Be clear** about your issue
- **Be patient** - maintainers are volunteers or have limited time
- **Be helpful** - share solutions you find to help others
- **Search first** - avoid duplicate questions
