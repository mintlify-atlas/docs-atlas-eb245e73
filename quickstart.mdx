## Get your first agent running in 3 steps

This guide will help you install 1Code, create your first chat session, and run an AI coding agent on your local project.

<Steps>
  <Step title="Download and install 1Code">
    Choose your installation method:

    <Tabs>
      <Tab title="Download (Recommended)">
        Visit [1code.dev](https://1code.dev) and download the pre-built app for your platform:
        
        - **macOS**: Download the DMG (Apple Silicon or Intel)
        - **Windows**: Download the installer (coming soon)
        - **Linux**: Download AppImage or DEB (coming soon)

        <Note>
          Pre-built releases include automatic updates and support for background agents. Your subscription helps maintain and improve 1Code.
        </Note>

        **macOS Installation:**
        1. Open the downloaded DMG file
        2. Drag 1Code to your Applications folder
        3. Launch 1Code from Applications
        4. If you see a security warning, go to **System Preferences** → **Security & Privacy** and click **Open Anyway**
      </Tab>
      
      <Tab title="Build from Source">
        Build 1Code yourself from the open-source repository:

        **Prerequisites:**
        - [Bun](https://bun.sh) runtime
        - Python 3.11+ with `setuptools` installed
        - macOS: Xcode Command Line Tools
        - Linux: Build essentials

        **Build steps:**
        ```bash
        # Clone the repository
        git clone https://github.com/21st-dev/1code.git
        cd 1code

        # Install dependencies
        bun install

        # Download required agent binaries (critical!)
        bun run claude:download
        bun run codex:download

        # Build the app
        bun run build

        # Package for your platform
        bun run package:mac     # macOS
        # bun run package:win   # Windows
        # bun run package:linux # Linux
        ```

        <Warning>
          Don't skip the `claude:download` and `codex:download` steps! These download the required agent binaries. Without them, the app will build but agents won't work.
        </Warning>

        The packaged app will be in the `release/` directory.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Create your first chat session">
    When you first launch 1Code, you'll be prompted to:

    1. **Select a project folder** - Choose any local Git repository or project directory
    2. **Configure your API keys** - Add your Anthropic API key for Claude, or OpenAI key for Codex
       - Click **Settings** → **API Keys**
       - Enter your API key and save

    <Tip>
      Don't have an API key? Get one from [Anthropic](https://console.anthropic.com/) or [OpenAI](https://platform.openai.com/).
    </Tip>

    3. **Start a new chat** - Click the **New Chat** button or press `Cmd/Ctrl + N`
  </Step>

  <Step title="Run your first agent">
    Now let's give your agent a task!

    **Try these example prompts:**

    <CodeGroup>
    ```text Simple task
    Add a README file with project setup instructions
    ```

    ```text Code review
    Review the authentication code in src/auth/ and suggest improvements
    ```

    ```text Bug fix
    Fix the TypeScript errors in the user profile component
    ```

    ```text Feature request
    Add dark mode support to the UI using CSS variables
    ```
    </CodeGroup>

    **Watch your agent work:**
    - See real-time tool execution (bash commands, file edits)
    - Review diffs before they're applied
    - View file changes with syntax highlighting
    - Approve or reject changes

    <Note>
      Each chat runs in its own Git worktree by default, keeping your main branch safe. You can disable this in settings if you prefer.
    </Note>
  </Step>
</Steps>

## Common first-run issues

<AccordionGroup>
  <Accordion title="Agent binary not found" icon="triangle-exclamation">
    **Error:** "Claude binary not found" or "Codex binary not found"

    **Solution:** If you built from source, make sure you ran the download scripts:
    ```bash
    bun run claude:download
    bun run codex:download
    ```

    The binaries should be in `resources/bin/` after download.
  </Accordion>

  <Accordion title="API key not working" icon="key">
    **Error:** "Invalid API key" or authentication failures

    **Solution:**
    1. Verify your API key is correct in **Settings** → **API Keys**
    2. Check your API key has sufficient credits/quota
    3. For Anthropic, ensure you're using a valid Claude API key (not Console key)
  </Accordion>

  <Accordion title="macOS security warning" icon="shield">
    **Error:** "1Code can't be opened because it is from an unidentified developer"

    **Solution:**
    1. Go to **System Preferences** → **Security & Privacy**
    2. Click **Open Anyway** next to the 1Code warning
    3. Confirm you want to open the app

    For builds from source, you may need to remove quarantine:
    ```bash
    xattr -cr /Applications/1Code.app
    ```
  </Accordion>

  <Accordion title="Python build errors" icon="python">
    **Error:** Build fails with Python-related errors

    **Solution:**
    1. Ensure Python 3.11+ is installed: `python --version`
    2. Install setuptools: `pip install setuptools`
    3. On Python 3.12+, setuptools is required for native module rebuilds
  </Accordion>
</AccordionGroup>
